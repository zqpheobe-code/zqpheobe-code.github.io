AI设计的发明能申请专利吗？——从一篇NAR顶刊论文说起

AI专利 深度学习 合成生物学 PCT CN/US/EPO

2024年，我和合作者在 Nucleic Acids Research（IF 16.7）上发表了一篇论文，用一个可解释的深度学习框架设计了具有跨物种活性的合成增强子（DOI: 10.1093/nar/gkae912）。

论文发表后，一个问题自然浮现：这些由AI设计的DNA序列，能不能申请专利？

我既是这篇论文的作者，也是一名涉外专利代理师（CN/US/EPO/JP/KR）。这篇文章我尝试从研究者和技术代理师两个视角，拆解"AI设计的发明"在专利法上到底面临哪些问题。

一、先搞清概念：AI辅助 vs AI自主

讨论"AI发明的专利问题"，首先要区分两种情况：

  • AI辅助发明：人提出问题、设计方向，AI作为工具生成具体方案，人做筛选和验证。我的NAR论文就属于这一类——我们设计了深度学习框架，用AI预测和生成增强子序列，但最终经过实验验证和筛选的是人。
  • AI自主发明：AI独立完成发明创造，人没有实质性的技术贡献。DABUS系列案件就是典型代表。

这两种情况在专利法上的待遇完全不同。

关键区分
目前全球主要专利局（CNIPA、USPTO、EPO）的共识是：AI不能作为发明人。但"AI作为工具辅助人类发明"是完全可专利的——这和你用Excel算数据、用CAD画图纸没有本质区别。

二、发明人是谁？——三局态度对比

美国（USPTO）

2022年，美国联邦巡回上诉法院在 Thaler v. Vidal 案中明确裁定："发明人"必须是人类（inventor必须是natural person）。2024年，USPTO发布了《AI辅助发明的发明人身份指南》，进一步确认：

  • AI不能被列为发明人
  • 但如果一个人对发明做出了"显著贡献"（significant contribution），可以作为发明人
  • 仅仅用AI跑了一下、没有实质性判断和改进，可能不够格

中国（CNIPA）

中国《专利法》第十三条规定发明人是指"对发明创造的实质性特点作出创造性贡献的人"。2023年修订的《专利法实施细则》没有直接提及AI，但实务中：

  • CNIPA不接受AI作为发明人（DABUS在中国的申请也被驳回）
  • AI辅助发明中，实际做出创造性贡献的人类研究者可以列为发明人
  • 没有"AI必须披露"的强制要求（这一点和美国2024指南的导向类似）

欧洲（EPO）

EPO在DABUS案中同样裁定：欧洲专利公约（EPC）第81条要求的发明人必须是自然人。EPO还要求申请人提交声明说明发明人是如何获得权利的。

但EPO的态度更务实：不关心你用了什么工具，只关心发明人是不是人。只要发明人栏填的是自然人，且该人确实做出了创造性贡献，EPO不会因为你在过程中用了AI就拒绝授权。

三、回到我的论文：能申请什么专利？

在NAR这篇论文中，我们做了一件事：训练了一个可解释的深度学习模型，用它来设计合成增强子，并在多个物种中验证了这些增强子的活性。

如果要把这个成果申请专利，可专利的客体包括：

  1. 方法权利要求：使用该深度学习框架设计增强子的方法（包括模型架构、训练数据、预测流程）
  2. 产品权利要求：通过该方法获得的特定合成增强子序列（需要具体序列，不能只写"AI设计的增强子"）
  3. 用途权利要求：这些合成增强子在基因工程/合成生物学中的应用
  4. 系统权利要求：包含该模型的计算机系统/装置

需要特别注意的三个问题

1. 充分公开（Written Description / Enablement）

专利说明书必须让本领域技术人员能够实现发明。如果AI模型是发明的核心，说明书中至少需要：

  • 描述模型的基本架构（不能只说"用深度学习"）
  • 提供足够的训练数据和参数信息使结果可复现
  • 如果模型权重是关键，可能需要部分公开或提供获取途径

这一点在USPTO特别严格——2023年 Amgen v. Sanofi 案后，美国对功能性权利要求的enablement要求大幅提高。如果你的权利要求写的是"通过AI设计的所有增强子"，但没有提供足够多具体序列，很可能因为enablement不充分被驳回。

2. 新颖性和创造性

AI设计的序列天然面临一个挑战：它"显而易见"吗？

如果审查员认为"任何人用类似的AI模型都能设计出这样的序列"，可能会以缺乏创造性为由驳回。你需要证明：

  • 模型本身的设计有创造性（比如我们的框架是"可解释的"，这一点区别于黑盒模型）
  • 或者，设计出的序列有出人意料的技术效果（比如跨物种活性，这是非显而易见的）

3. 披露AI使用的义务

目前，CNIPA、USPTO、EPO都没有强制要求申请人在专利文件中披露AI的使用情况。但USPTO 2024年的指南暗示，如果AI使用与发明人身份认定相关，申请人可能需要向USPTO说明。

实务建议：主动披露比隐瞒更安全。在说明书中如实描述AI的使用方式，既能满足充分公开的要求，也能在发明人身份问题上避免后续争议。

四、如果走PCT，怎么布局？

假设你想就AI设计增强子的方法申请国际专利，PCT是一个合理选择。策略上需要注意：

  1. 优先权：在中国首次申请后12个月内提交PCT。如果论文已经发表，必须在发表前提交首次申请（否则丧失新颖性）。
  2. 权利要求层次：独立权利要求写方法（最宽），从属权利要求逐步收窄到具体序列。产品权利要求单独布局。
  3. 进入国家阶段的选择：生物技术领域，US（最大市场）、EP（统一保护）、JP（合成生物学强国）、KR（生物制药活跃）是重点。
  4. 各国差异：
    • US：对软件/AI相关方法权利要求有Alice两步法审查（35 U.S.C. § 101），需要证明不是"抽象概念"
    • EPO：技术特征必须达到"technical character"要求，纯数学方法不行，但"用数学方法控制技术系统"可以
    • 中国：审查指南2023修订版已纳入AI相关审查规则，要求技术方案解决技术问题

五、总结：四个实操建议

给研究者和企业的建议
  1. 论文发表前先评估专利。一旦论文公开，新颖性就没了。如果你用AI设计了有商业价值的序列，在投稿前先提交专利申请（哪怕只是 provisional application）。
  2. 发明人栏只填人类。不管AI做了多少工作，发明人必须是做出"创造性贡献"的自然人。在论文中，对实验设计、数据分析、结论判断有实质贡献的人才有资格。
  3. 说明书中如实描述AI的使用。这不是法律强制要求，但能避免后续发明人资格争议，同时满足充分公开。
  4. 权利要求要有层次。方法→产品→用途，从宽到窄。AI相关的方法权利要求在各局都可能面临客体审查挑战，备好具体序列作为退路。

写在最后

AI不会取代发明人，但会改变发明的方式。作为同时活跃在AI研究和专利实务两端的人，我持续关注这个交叉领域的法律演进。如果你有AI相关发明的专利问题，欢迎交流。
